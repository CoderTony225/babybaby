## ⚠️ 常见问题解决

### 问题 1：后端启动失败 - "拒绝访问"
**症状：** 
```
ERROR: C:\Users\Tony0\Desktop\cupro\baseplatform\target\classes\application.yml (拒绝访问)
```

**解决方案：**
1. 关闭所有正在运行的后端进程
2. 手动删除 `target` 目录
3. 重新运行 `启动后端.bat`

或者在 PowerShell 执行：
```powershell
cd c:\Users\Tony0\Desktop\cupro\baseplatform
Remove-Item -Recurse -Force target -ErrorAction SilentlyContinue
.\mvnw.cmd clean spring-boot:run
```

### 问题 2：前端启动失败 - "EPERM: operation not permitted"
**症状：**
```
Error: EPERM: operation not permitted, mkdir 'node_modules\.vite\deps_temp'
```

**解决方案：**
1. 关闭所有正在运行的前端进程
2. 删除 `node_modules\.vite` 目录
3. 重新运行 `启动前端.bat`

或者在 PowerShell 执行：
```powershell
cd c:\Users\Tony0\Desktop\cupro\frontend
Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue
npm run dev
```

### 问题 3：端口被占用
**症状：**
```
Port 8080 was already in use
或
Port 5173 was already in use
```

**解决方案：**
1. 找到并关闭占用端口的进程：
```powershell
# 查找占用 8080 端口的进程
netstat -ano | findstr :8080
# 查找占用 5173 端口的进程
netstat -ano | findstr :5173
```

2. 杀死进程（替换 PID）：
```powershell
taskkill /F /PID <进程 ID>
```

3. 或者修改端口：
   - 后端：修改 `application.yml` 中的 `server.port`
   - 前端：修改 `vite.config.js` 中的 `server.port`

### 问题 4：npm 依赖安装失败
**症状：**
```
npm ERR! ERESOLVE could not resolve
```

**解决方案：**
```powershell
cd c:\Users\Tony0\Desktop\cupro\frontend
npm install --legacy-peer-deps
```

## 🛠️ 停止服务

### 方法 1：命令行启动的服务
- 在运行服务的命令行窗口按 `Ctrl + C`

### 方法 2：脚本启动的服务
- 关闭对应的命令行窗口

### 方法 3：强制停止
```powershell
# 停止所有 Java 进程（后端）
taskkill /F /IM java.exe

# 停止所有 Node 进程（前端）
taskkill /F /IM node.exe
```
