## 功能背景

随着无人机资产增多，需要统一的电子化台账支撑录入、查询、维护与检索。人工维护规格参数易出错且成本高，希望通过 AI 根据用户提供的简要输入自动生成结构化属性，降低录入负担并提高一致性。

**当前痛点**：

- 无人机规格字段多，手工逐项填写耗时长、易不一致。
- 缺少统一的查询与修改入口，资产信息分散。
- 首期需在单机/轻量环境快速落地，后续再平滑切换到集中式数据库。

**预期收益**：

- 单次录入在完成校验的前提下，主要字段可由 AI 辅助生成，减少重复劳动。
- 核心 CRUD 与检索在同一模块内完成，便于运维与审计。
- 支持 SQLite 与 MySQL 切换，兼顾开发便携与生产扩展。

---

## 目标

**主要目标**：

1. 提供无人机信息的新增、分页查询、详情查看、修改、删除能力；列表支持关键字检索（至少覆盖编号、型号、备注）。
2. 在用户提交创建或编辑时，基于约定输入调用 AI 服务生成扩展属性（结构化存储）；AI 失败时允许按策略降级（保存人工已填字段并提示），具体策略见下文验收标准。
3. 数据访问层与服务层均采用「接口与实现分离」的包结构；HTTP 请求经过统一拦截器记录可追溯的请求摘要日志。

**成功指标**：

- 核心 CRUD 与列表检索验收用例全部通过。
- 在约定 AI 超时时间内，成功率与降级行为符合验收标准（可在测试环境通过 Mock 验证）。
- 通过切换 Spring Profile / 配置可在 SQLite 与 MySQL 间部署同一套制品（初期默认 SQLite）。

## 用户故事

### 故事 1：录入无人机并生成 AI 属性

**As a** 已登录操作员  
**I want** 填写必要信息后保存无人机档案，并由 AI 自动生成扩展属性  
**So that** 减少手工录入规格字段的时间并保持一致性

#### 验收标准

- **Given** 操作员已登录且具备「无人机-新增」权限  
  **When** 填写必填项（至少包含机身编号、型号名称）并提交保存  
  **Then** 系统调用 AI 服务生成扩展属性并持久化，成功后跳转列表或详情页并提示成功

- **Given** AI 服务在约定超时时间内不可用或返回错误  
  **When** 操作员提交保存  
  **Then** 系统保存用户已填写字段（不含或部分不含 AI 字段），页面展示明确错误/告警信息，并允许后续在编辑页重新触发 AI 生成（若设计启用）

#### 异常场景

- **Given** 机身编号与已有档案重复  
  **When** 提交保存  
  **Then** 拒绝保存并提示重复，不调用或中断后续持久化

- **Given** 必填项缺失或格式非法  
  **When** 提交保存  
  **Then** 拒绝保存并在表单展示校验错误，不调用 AI

---

### 故事 2：查询与分页浏览

**As a** 已登录操作员  
**I want** 按关键字与分页条件浏览无人机列表  
**So that** 快速定位目标设备

#### 验收标准

- **Given** 列表页已打开  
  **When** 输入关键字并查询  
  **Then** 结果集仅包含编号、型号或备注等约定字段匹配的记录（大小写规则以实现为准），并支持分页导航

- **Given** 无匹配记录  
  **When** 查询  
  **Then** 展示空列表状态，不报错

---

### 故事 3：修改与删除

**As a** 已登录管理员或具备编辑权限的操作员  
**I want** 修改或删除无人机档案  
**So that** 台账信息与实际情况一致

#### 验收标准

- **Given** 目标记录存在且当前用户具备编辑权限  
  **When** 保存修改  
  **Then** 更新数据库记录并展示成功提示；若启用「编辑时重新 AI 生成」，行为与故事 1 一致

- **Given** 目标记录存在且当前用户具备删除权限  
  **When** 确认删除  
  **Then** 记录被删除（物理或逻辑删除以实现设计为准），列表不再展示该记录

## 非功能性需求

### 性能要求

- **响应时间**：列表分页与详情查询在常规数据量（例如万级以内）下页面可接受；AI 调用单独配置超时（建议默认 10～30 秒可配置），避免阻塞 servlet 线程过久（设计上可考虑异步或独立线程池，具体以实现为准）。
- **并发量**：首期目标为中小团队并发（例如数十并发），不做极端高并发承诺。
- **数据量**：单表建议支撑至少数万条无人机记录仍可分页查询。

### 安全要求

- **认证**：采用 Apache Shiro，未登录用户不可访问业务页与敏感接口。
- **授权**：无人机模块的新增、编辑、删除、查询按角色或权限串细分（细则在设计文档中落地）。
- **数据保护**：数据库连接密码、AI API Key 等置于配置文件或环境变量，禁止写入仓库；日志中对 Authorization、Cookie、Token、API Key 脱敏。
- **输入验证**：服务端使用 Hibernate Validator 校验；持久层使用参数绑定防止 SQL 注入。

### 兼容性要求

- **浏览器**：支持 Chrome、Edge、Firefox 当前主流版本；Bootstrap 3 页面在 IE11 如需支持须在设计与测试中单独声明（本期默认不强制 IE11，如需请立项补充）。
- **数据库**：同一套应用通过配置在 **SQLite（默认）** 与 **MySQL** 间切换；SQL 方言差异在设计中集中处理。
- **运行栈**：与项目约定的 Java EE 8 / Servlet 3.0、Spring Boot 2.2.x、Spring 5.2.x、Shiro 1.7、MyBatis 3.5.x、Thymeleaf 3.0.x、Bootstrap 3.3.7 保持一致。

### 可用性要求

- **故障恢复**：数据库连接失败时给出可读错误页或提示；AI 不可用时不导致核心表单数据无法保存（遵循降级策略）。

## 排除范围

本次迭代**不包含**以下内容：

- 飞控实时遥测、地图航迹、飞行任务调度等非台账类功能。
- 移动端独立 App、微信小程序。
- 批量 Excel 导入/导出（可作为后续迭代）。
- 多租户隔离与跨组织数据权限（除非单独需求确认）。
- 对接具体厂商无人机云平台的自动同步。

**后续迭代计划**：

- 批量导入导出、图片/附件上传、审计日志查询、与外部资产管理系统的接口。
